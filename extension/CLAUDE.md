# Extension — A11yTrail (Manifest V3, TypeScript + Vite/CRXJS)

A **standalone, fully client-side** accessibility checker. Click the toolbar icon → the popup
injects **bundled axe-core** into the current tab via `chrome.scripting`, runs the scan **in
the page**, and shows a score, severity breakdown, sorted issues with fix guidance, highlight,
copy-for-ticket, and HTML/JSON export. **No backend, no network requests, no data collection.**

This is the build we ship to the Chrome Web Store first. The previous backend-coupled client
(login + remote scan) is **quarantined, not deleted**, in `backend-client-wip/` and will be
revived for continuous monitoring **after** store approval. See `backend-client-wip/README.md`.

## Layout
- `src/lib/` — pure, unit-tested core: `normalize` (axe → model), `severity`, `score`,
  `wcag` (tags → level/label/filter), `fixmap` (curated remediation + axe fallback),
  `format` (copy-for-ticket + HTML/JSON export), `escape`, `types`.
- `src/scan/inject.ts` — `chrome.scripting` glue: inject bundled axe, run it, highlight/clear.
  The injected functions are **self-contained** (params + page globals only) — chrome
  serializes them with `.toString()`, so they must not reference module scope.
- `src/popup/` — `index.html` + `popup.css` + `popup.ts` (orchestration) + `render.ts`
  (pure, tested HTML builders).
- `public/axe.min.js` — vendored axe-core (gitignored; regenerated by `copy-axe`).
- `icons/` — generated PNG icons (`npm run icons`).

## Commands
- `npm install` · `npm run dev` · `npm run build` (→ `dist/`; runs `copy-axe` first)
- `npm test` (Vitest — `src/lib` + `src/popup/render`) · `npm run typecheck`
- `npm run package` (build + zip `dist/` → store upload) · see `STORE_SUBMISSION.md`
- Reload the unpacked `dist/` at `chrome://extensions` after building.

## Policy guardrails (these gate publishing — keep them true)
- Permissions stay exactly `["activeTab","scripting"]`. No `host_permissions`, `tabs`,
  `storage`, `downloads`, `<all_urls>`, or `web_accessible_resources`.
- axe-core is **bundled** and injected from the package — never fetched from a CDN at runtime.
- No inline scripts / no `eval` (CSP-clean). Export downloads use a Blob + `<a download>`,
  not the `downloads` permission.
- Zero network requests, zero data collection. Highlight-only — no auto-fix/overlay product.
- The popup must pass its own accessibility bar (focus styles, keyboard nav, ARIA, contrast).

## MV3 gotchas
- `activeTab` grants the current tab only on user invocation (popup open) — enough to read
  `tab.url` and inject; no background service worker is needed for this scanner.
- Content/injected scripts run in the isolated world; axe injected via `files` defines
  `window.axe` there, and the subsequent `func` injection in the same frame sees it.
